# Deployment Detailed Steps: AI-Generated React + Tailwind + Vite App to Vercel

## 1. Purpose of This Guide

This document provides detailed deployment steps for converting an AI-generated React `.tsx` file into a working Vite React application and deploying it to Vercel using GitHub.

Use this guide when an AI tool such as Gemini Canvas, Claude, ChatGPT, or another app builder gives you a React component file rather than a complete deployable project.

---

## 2. Standard Deployment Flow

```text
AI-generated .tsx file
        ↓
Create Vite React TypeScript project
        ↓
Copy code into App.tsx
        ↓
Install missing dependencies
        ↓
Configure Tailwind CSS
        ↓
Test locally
        ↓
Run production build
        ↓
Commit with Git
        ↓
Push to GitHub
        ↓
Deploy with Vercel
```

---

## 3. Prerequisites

Before starting, confirm these tools are installed.

### 3.1 Check Node.js and npm

Open Command Prompt and run:

```cmd
node -v
npm -v
```

Expected result:

```text
Version numbers should appear for both commands.
```

If `node` is not recognised, install Node.js and reopen Command Prompt.

### 3.2 Check Git

Run:

```cmd
git --version
```

Expected result:

```text
git version x.x.x
```

If `git` is not recognised, install Git for Windows and reopen Command Prompt.

### 3.3 Required Online Accounts

You need:

- A GitHub account
- A Vercel account

Using the same GitHub account to sign into Vercel makes deployment easier.

## 9. Configure Tailwind CSS

If the page loads but looks unstyled or different from the AI preview, Tailwind CSS is probably missing.

Install Tailwind for Vite:

```cmd
npm install tailwindcss @tailwindcss/vite
```

Open:

```text
vite.config.ts
```

Replace the content with:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

Open:

```text
src\index.css
```

Replace all content with:

```css
@import "tailwindcss";
```

Restart the development server:

```cmd
npm run dev
```

Open:

```text
http://localhost:5173/
```

The app should now match the AI-generated design more closely.

## 10. Add Static Images

Put static images in the `public` folder.

Example:

```text
public\ClaudeCodeTutorial.png
```

In React, reference it as:

```tsx
src="/ClaudeCodeTutorial.png"
```

Do not use the full Windows path inside React.

Incorrect:

```tsx
src="C:\Professional Documents\Everything AI\Claude\ClaudeCodeTutorial\public\ClaudeCodeTutorial.png"
```

```tsx
src="/ClaudeCodeTutorial.png"
```

## 13. Fix TypeScript Build Errors

If the local app works but `npm run build` fails with many TypeScript errors, the generated `.tsx` file may not be strictly typed.

Quick fix for first deployment:

Open:

```text
src\App.tsx
```

Add this line at the very top:

```tsx
// @ts-nocheck
```

Example:

```tsx
// @ts-nocheck
import React from "react";
```

Run the build again:

```cmd
npm run build
```

Long-term improvement:

- Remove `@ts-nocheck`
- Fix TypeScript errors properly
- Add interfaces and types

## 15. Create a GitHub Repository

Go to GitHub and create a new repository.

Recommended name format:

```text
project-name-tutorial
```

Example:

```text
claudecode-tutorial
```

Important:

Do not select:

- Add README
- Add `.gitignore`
- Choose a license

The GitHub repository should be empty because your local project already has files.

Copy the HTTPS repository URL.

Example:

```text
https://github.com/your-username/claudecode-tutorial.git
```

## 16. Push Local Code to GitHub

Connect the local project to GitHub:

```cmd
git remote add origin https://github.com/your-username/claudecode-tutorial.git
```

Rename the branch to `main`:

```cmd
git branch -M main
```

Push:

```cmd
git push -u origin main
```

Expected:

```text
branch 'main' set up to track 'origin/main'
```

Refresh GitHub and verify the project files are visible.

---

## 17. Deploy to Vercel

Open:

```text
https://vercel.com
```

Sign in using GitHub if possible.

Click:

```text
Add New → Project
```

Import the GitHub repository.

Vercel should auto-detect Vite.

Confirm settings:

```text
Framework Preset: Vite
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

Click:

```text
Deploy
```

Wait for deployment to finish.

Expected status:

```text
Ready
```

Open the live URL.

Example:

```text
https://claudecode-tutorial.vercel.app
```

## 19. How to Reuse These Markdown Files via GitHub

### 19.1 Add Them to the Current Project

Create a folder:

```text
docs
```

Place these files inside it:

```text
docs/Purpose_and_Overview_AI_React_Vercel_Deployment.md
docs/Deployment_Detailed_Steps_AI_React_Vercel.md
```

Commit and push:

```cmd
git add docs
git commit -m "Add deployment markdown documentation"
git push
```

Now the documentation is available directly inside the GitHub repository.

---

### 19.2 Reuse in a New Project

For a future project:

1. Create or clone the new project repository.
2. Copy the `docs` folder from the previous repository.
3. Update project-specific values:
   - Project name
   - Local path
   - GitHub URL
   - Vercel URL
   - Image filenames
   - Required packages
4. Commit the updated documentation.

Commands:

```cmd
git add docs
git commit -m "Add reusable deployment documentation"
git push
```

---

### 19.3 Use a Central Documentation Repository

Create a separate GitHub repository such as:

```text
ai-react-vercel-deployment-playbook
```

Store the Markdown files there.

For every future project, copy the latest version from that repository.

This gives you one central reusable source.

---

### 19.4 Use a GitHub Template Repository

For the best long-term approach, create a template repository with:

```text
Vite React TypeScript setup
Tailwind CSS setup
Standard docs folder
Placeholder App.tsx
Placeholder public image
README.md
```

For every new AI-generated project:

1. Create a new repository from the template.
2. Replace `src/App.tsx` with the AI-generated `.tsx` content.
3. Add images to `public`.
4. Install any missing dependencies.
5. Test and deploy.

This is the cleanest reuse method because it avoids repeating setup steps.

## 20. Common Issues and Fixes

### Issue 1: `node` is not recognised

Fix:

```text
Install Node.js and reopen Command Prompt.
```

---

### Issue 2: `npm.ps1 cannot be loaded`

Fix:

```text
Use Command Prompt instead of PowerShell.
```

---

### Issue 3: `git` is not recognised

Fix:

```text
Install Git for Windows and reopen Command Prompt.
```

---

### Issue 4: Tailwind styles do not appear

Fix:

```cmd
npm install tailwindcss @tailwindcss/vite
```

Check:

```text
vite.config.ts
src/index.css
```

---

### Issue 5: Missing package error

Example:

```text
Failed to resolve import "lucide-react"
```

Fix:

```cmd
npm install lucide-react
```

For other packages:

```cmd
npm install package-name
```

---

### Issue 6: Build fails but local preview works

Fix:

```cmd
npm run build
```

Read the error.

For generated TypeScript issues, quick fix:

```tsx
// @ts-nocheck
```

at the top of `src/App.tsx`.

---

### Issue 7: Image does not appear

Check:

- Image is inside `public`
- Filename is correct
- Extension is correct
- Path begins with `/`

Correct example:

```tsx
src="/ClaudeCodeTutorial.png"
```

---

### Issue 8: Vercel deployment fails

Run locally first:

```cmd
npm run build
```

If the local build fails, the Vercel build will also fail.

Fix the local build, then push again:

```cmd
git add .
git commit -m "Fix build issue"
git push
```

---

## 21. Best Practice Checklist

Before pushing to GitHub:

- Local site runs correctly
- Tailwind styling works
- Images load
- Buttons work
- Mobile layout is acceptable
- `npm run build` succeeds
- Commit message is meaningful

Before considering deployment complete:

- Vercel status is `Ready`
- Public URL opens correctly
- Browser hard refresh works
- Site works in another browser or mobile device

---

## 22. Key Principle

Do not treat Vercel as the place to edit the app.

Use this model:

```text
Local project = where you edit
GitHub = where you store and version the code
Vercel = where the website is hosted
```

This keeps deployments clean, repeatable, and reliable.
